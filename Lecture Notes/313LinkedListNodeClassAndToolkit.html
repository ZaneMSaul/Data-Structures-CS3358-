Linked List Node Class and Toolkit

In Lecture Note 312LinkedListConceptualIntro, the fundamental issues of linked lists are discussed using a node specification based on C-struct (that allows us to package the data field and link field of an item of a linked list, called Node, in a minimal way – the Node data type specified as such consists of only data members and a Node object's data members can be freely read and/or changed by any function to which the object is visible). Doing so minimizes the need to deal with implementation details (that are language related rather than data structure related) and allows the discussion to be better focussed on aspects that are data structure related. However, as alluded to in 312LinkedListConceptualIntro, after having understood the linked list data structure, we would ultimately want to learn how to implement and apply the data structure using the features of our chosen language (C++) that offer the best support (class, etc.). Along this line, we will augment our study of linked lists with some C++ implementational considerations by reviewing the approach and code discussed in Chapter 5 of the textbook.

Rationale behind toolkit

In using the node class to implement the bag ADT using linked list as shown in the preceding section, some functions implementing common operations for manipulating a linked list are written to facilitate the implementation of the various bag operations. These common linked list manipulation functions are specifically written for the bag class and are included as non-member functions of the class. Now, if we also want to use the same node class to implement yet other ADTs (such as the sequence ADT) with linked list, we will most likely find the same set of common linked list manipulation functions useful. We can then reuse the set of common linked list manipulation functions (in the same fashion as we did implementing the bag ADT) by simply reproducing them in the other ADTs. However, a better and more elegant approach is to make the set of common linked list manipulation functions into a separate function library that requires the specification and implementation of the functions to be written only once (instead of being reproduced over and over). Such a function library is referred to by the authors of the textbook as a toolkit. Moreover, because the linked list toolkit (library of common linked list manipulation functions) is very closely related to the node class (since the linked lists that the functions operate on are constructed from instances of the node class), the authors have elected to combine the specifications for the node class and the toolkit functions into one specification; fittingly, the implementations for the node class and the toolkit functions are also combined into one implementation.
